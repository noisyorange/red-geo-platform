# 问题分析文档 - 多Excel数据日期筛选问题

## 问题描述
同一个项目下上传了多个Excel文件（如2月27日和2月28日的数据），但Dashboard的日期筛选下拉框只能看到其中一个日期（2月27日），新上传的日期（2月28日）无法选择。

## 问题分析

### 数据流程
1. 运营人员在数据上传页面上传Excel文件
2. 每上传一个文件，会在 `project_data` 表中创建一条新记录
3. Dashboard 加载数据时，会查询该项目的所有 `project_data` 记录
4. 将所有记录的 data 字段合并到 excelData 中
5. 通过 query_time 字段去重，生成日期筛选下拉框

### 可能原因
1. **日期格式不一致**：2月28日的日期格式可能与2月27日不同，导致正则过滤后被排除
2. **数据未正确保存**：新上传的数据可能没有正确保存到数据库
3. **数据合并问题**：多条数据的合并逻辑可能有问题
4. **日期格式正则过滤问题**：当前正则可能无法匹配所有日期格式

### 当前代码逻辑 (Dashboard.tsx)
```typescript
const queryTimes = useMemo(() => {
  const times = [...new Set(excelData.map(d => d.query_time))];
  // 当前正则：只匹配特定格式
  const validTimes = times.filter(t => t && 
    (t.match(/^\d{4}[-/]\d{1,2}[-/]\d{1,2}$/) || 
     t.match(/^\d{1,2}[-/]\d{1,2}[-/]\d{4}$/))
  );
  return validTimes.sort();
}, [excelData]);
```

### 当前支持的日期格式
- `2025/2/27` - 年/月/日
- `2025-2-27` - 年-月-日
- `27/2/2025` - 日/月/年
- `27-2-2025` - 日-月-年

### 不支持的日期格式（可能存在的问题）
- `2025/02/27` - 带前导零的月份
- `2025/2/27 12:00:00` - 包含时间
- `2025.2.27` - 使用点分隔
- 中文格式

## 排查步骤

### 1. 检查数据库中的数据
需要确认 `project_data` 表中是否存在多条记录，以及每条记录的 `data` 字段中 `query_time` 的格式。

### 2. 检查前端加载的数据
在Dashboard中添加调试日志，检查：
- excelData 中的所有 query_time 值
- 过滤后的 validTimes 值

### 3. 可能的解决方案
- 放宽日期格式正则匹配
- 或者不在前端过滤，由后端返回原始数据
