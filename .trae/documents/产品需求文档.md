# 小红书 GEO 服务监控工具 - 产品需求文档

## 1. 项目概述

**项目名称**：小红书 GEO 服务监控工具  
**项目类型**：Web 管理系统  
**部署环境**：海外阿里云服务器  
**核心功能**：为运营提供售前表单管理和数据上传功能，为客户提供品牌 Query 监控看板  
**目标用户**：运营人员、品牌客户

---

## 2. 角色定义

| 角色 | 权限描述 |
|------|----------|
| 运营 (Admin) | 填写 8 模块售前表单、上传 Excel 数据、管理客户 |
| 客户 (Customer) | 登录后查看自己品牌的 Query 监控看板（Top 3 推荐） |

---

## 3. 核心功能模块

### 3.1 登录模块 (/login)
- 用户名/密码登录
- 角色区分：运营/客户
- 登录后根据角色跳转不同页面

### 3.2 运营后台

#### 3.2.1 售前表单 (/admin/onboarding)
8 个模块的售前表单：
1. **品牌基本信息** - 品牌名称、行业、规模
2. **目标市场** - 目标国家/地区
3. **竞争对手分析** - 主要竞争对手列表
4. **关键词策略** - 核心关键词、 长尾关键词
5. **内容策略** - 内容类型、发布频率
6. **投放预算** - 预算范围、投放周期
7. **KPI 目标** - 曝光量、互动率、转化目标
8. **特殊需求** - 其他特殊要求

#### 3.2.2 数据上传 (/admin/upload)
- Excel 文件上传
- 支持批量导入 Query 数据
- 数据预览和验证

### 3.3 客户看板 (/dashboard)
- 展示自己品牌的监控数据
- Top 3 推荐 Query
- 数据可视化展示

---

## 4. 页面路由

| 路由 | 页面 | 角色 |
|------|------|------|
| /login | 登录页 | 全部 |
| /admin/onboarding | 售前表单 | 运营 |
| /admin/upload | 数据上传 | 运营 |
| /dashboard | 监控看板 | 客户 |

---

## 5. 技术要求

- React + TypeScript 前端
- Express.js 后端
- 部署至海外阿里云服务器
- 支持响应式布局
